# A Fixed-Point View of Post-Hoc Temperature Calibration

This repo goes with a short paper (`paper/paper.tex`) about temperature
scaling, the usual post-hoc method for fixing miscalibrated neural net
probabilities. The idea: instead of just handing it to a generic
optimizer, I reformulated it as a fixed-point problem and solved it
with a Picard-style iteration, the kind of thing you'd normally see in
a numerical methods course for nonlinear equations or PDEs.

Three solvers for the temperature parameter, compared against each other:

| Method | Idea | Convergence |
|---|---|---|
| `fit_temperature_optimizer` | Standard scalar minimization of NLL (Guo et al., 2017) | Reference / baseline |
| `fit_temperature_picard` | Naive multiplicative fixed-point (iterative-scaling style) update | Linear, slow |
| `fit_temperature_steffensen` | Same fixed-point map + Steffensen/Aitken acceleration | Quadratic, fast |

All three land on the same temperature. The accelerated version gets
within machine precision of the baseline optimizer in about 6
iterations; the naive version needs hundreds.

## Layout

```
src/
  data.py          # loads and splits the sklearn digits dataset
  model.py         # a small NumPy MLP
  calibration.py   # ECE/MCE metrics + the three temperature solvers
  experiment.py    # trains, calibrates, generates all figures/metrics
tests/
  test_calibration.py   # tests for the calibration module
  test_model.py          # tests for the MLP, incl. a gradient check
paper/
  paper.tex        # the write-up
  references.bib
  figures/         # generated by experiment.py
results/
  metrics.json     # generated by experiment.py
```

## Running it

```bash
pip install -r requirements.txt
python -m src.experiment   # regenerates paper/figures/*.png and results/metrics.json
python demo.py              # same thing, but narrated step by step
pytest tests/                # run the tests
```

To rebuild the PDF (needs a LaTeX install):

```bash
cd paper && latexmk -pdf paper.tex
```

## Why the digits dataset, not CIFAR / ImageNet

This project is about the calibration method, not about squeezing
accuracy out of a big vision model. sklearn's digits dataset keeps
everything reproducible offline, no downloads, while still being real
multi-class data with actual calibration error to fix.

## Status

This is a self-directed project, not a paper submitted anywhere. The
fixed-point framing and the Steffensen acceleration are, as far as I
know, not something already written up for temperature scaling
specifically, even though the underlying pieces (iterative scaling,
Steffensen's method) are both classical. Said explicitly in the paper's
intro too.
